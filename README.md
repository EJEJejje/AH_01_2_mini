이 저장소는 **2조 프로젝트 저장소**입니다.

---

## 브랜치(Branch) 규칙

- **main 브랜치**
  - 최종 확정 코드가 모이는 브랜치입니다.
  - 최소 **1명 이상 리뷰어 승인(approval)** 후에만 머지(Merge)합니다.
  - 테스트가 끝난 안정된 코드만 들어가야 합니다.

- **test 브랜치**
  - 기능 동작을 통합해서 확인하는 **통합 테스트용 브랜치**입니다.
  - 새 기능/모델/EDA는 `feat/...` 브랜치에서 작업한 뒤  
    **PR(Pull Request)를 test 브랜치로** 보냅니다.

- **fix 브랜치**
  - 버그 수정용 브랜치입니다.
  - 버그를 수정한 뒤 `fix/...` 브랜치에서 커밋하고  
    **PR을 test 브랜치로** 보냅니다.

- **save 브랜치**
  - 프로젝트 기간 동안 **중간 저장(backup)** 용도로 사용하는 브랜치입니다.
  - 실험 결과나 임시 코드 등을 보관할 때 사용합니다.
  - 보통 PR 없이 필요할 때만 머지하거나 푸시합니다.

---

## 작업 상황별 브랜치 전략

| 상황                         | 시작 브랜치               | 새 브랜치 이름 예시           | PR 대상 브랜치 |
|----------------------------|-------------------------|-----------------------------|---------------|
| 새 기능/모델/EDA 추가           | `test`                  | `feat/eda-본인이름`          | `test`        |
| 버그 수정                     | `test` 또는 `main`        | `fix/bug-설명`              | `test`        |
| 단순 백업(중간 저장)            | `save` 또는 자신의 `feat/...` | 필요 시 `save`로 merge      | 보통 PR 없음   |
